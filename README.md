# Transfer Examiner

Гибридная система оценки риска банковских переводов по 115-ФЗ: эвристики + LLM (GigaChat).

## Описание:
Данный проект оценивает риск банковского перевода с точки зрения 115-ФЗ (противодействие легализации доходов) на основе:
- Суммы перевода;
- Типа получателя (физлицо, ИП, юрлицо);
- Назначения платежа.

## Проблема:
По требованию законодательства и ЦБ банки обязаны контролировать денежные операции, чтобы предотвратить отмывание доходов и финансирование терроризма. Если перевод кажется подозрительным, банк может его приостановить и запросить подтверждающие документы. Данный процесс занимает много времени и ресурсов, в том числе человеческого.

## Решение:
Мой проект позволит банку в автоматическом режиме с помощью искусственного интеллекта на основании вводных данных проверить и оценить уровень риска по переводу, выведет причины решения и выдаст рекомендации по данной ситуации.

## Как работает

1. **Эвристики** — быстрые правила на основе опыта работы в банке (суммы, ключевые слова, типы получателей)
2. **LLM (GigaChat)** — для пограничных случаев: сумма от 400 000 до 1 000 000 рублей при риске medium/high
3. Если LLM недоступна — система возвращается к эвристикам

## Пример работы

**Вход:** сумма 500 000 руб., получатель "individual", назначение "Оплата по счету"

**Выход:**

```json
{
  "risk": "high",
  "reason": "Сумма перевода превышает обязательный контроль...",
  "recommendation": "Запросить подтверждающие документы...",
  "method": "llm"
}
```
